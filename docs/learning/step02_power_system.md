# Step 2: AR Walker 전원 시스템 이해

> 우리 프로젝트의 전기가 어디서 오고, 어디로 가는지 이해하자.

---

## 1. 전원 트리 (Power Tree)

AR Walker의 전원은 **48V 배터리 하나**에서 시작해서 여러 갈래로 나뉜다:

```
[48V 리튬 배터리]
    │
    ├──→ [Elmo Gold Twitter] → [모터] (48V 직접 사용)
    │
    └──→ [벅 컨버터 ①] (48V → 15V, 65W 이상)
              │
              └──→ [15V 출력] (12~19V 범위)
                      │
                      ├──→ [Jetson Orin Nano Super] (~40W)
                      │
                      ├──→ [캡처카드 + 카메라] (~20W)
                      │
                      └──→ [벅 컨버터 ②] (15V → 5V, 소형)
                                │
                                └──→ [5V 출력]
                                        │
                                        ├──→ [EBIMU RF 리시버] (5V)
                                        │
                                        └──→ [LDO]
                                                │
                                                └──→ [3.3V 출력]
                                                        │
                                                        ├──→ [STM32H743]
                                                        ├──→ [ESP32-S3]
                                                        ├──→ [INA828 x4]
                                                        └──→ [CAN 트랜시버]
```

> **직렬 구조 채택** — 48V → 15V → 5V → 3.3V
> - 벅 ① (48V→15V): 대전력 65W+ — Jetson, 캡처카드, 그리고 벅②의 입력
> - 벅 ② (15V→5V): 소전력 ~5W — 15V 입력이라 **흔하고 싼 부품** 사용 가능
> - LDO (5V→3.3V): MCU/센서 로직 전원
>
> **왜 병렬(48V→5V)이 아닌 직렬(15V→5V)인가?**
> - 48V→5V 벅은 고입력 전압 부품이 필요 → 비싸고 선택지 적음
> - 15V→5V 벅은 아무데서나 구할 수 있음 (아두이노 모듈급)
> - MCU 전력(~5W)이 작아서 효율 차이(~6%) 무시 가능
> - Jetson 전원 없으면 어차피 시스템 운영 불가 → 독립성 불필요

## 2. 벅 컨버터 ① — 48V → 15V (Jetson + 캡처카드)

### 이것이 가장 큰 전력 소비자!

Jetson Orin Nano Super는 **5V가 아니라 12~19V** 전원이 필요하다.
캡처카드(카메라용)도 같은 12~19V 범위로 전원을 공급해야 한다.

```
요구사항:
  입력: 48V (배터리)
  출력: 15V (12~19V 범위 내)
  전력: 65W 이상 (Jetson ~40W + 캡처카드 + 여유)
  전류: 65W / 15V = 4.3A 이상

→ 대전력 벅 컨버터 필요! (일반 소형 벅으로는 부족)
```

### 왜 15V인가?
```
Jetson Orin Nano Super 입력 범위: 12~19V
캡처카드 입력 범위: 12~19V

15V = 범위의 중간값 → 가장 안정적
  - 12V에 너무 가까우면: 부하 시 전압 강하로 12V 밑으로 떨어질 수 있음
  - 19V에 너무 가까우면: 발열 증가
```

## 3. 벅 컨버터 ② — 15V → 5V (MCU/센서)

### 뭐하는 놈인가?
- 벅①이 만든 15V를 **5V로 한 번 더 낮추는** 장치
- 입력이 15V라서 **흔하고 저렴한 부품**으로 가능

### 왜 15V에서 5V로?
- STM32, ESP32는 3.3V에서 동작
- 15V를 직접 LDO로 3.3V를 만들면: (15-3.3)×1A = **11.7W 열 발생** → 위험
- 15V → 5V (벅) → 3.3V (LDO) 으로 단계를 나눠야 발열 적음

### 동작 원리 (간단 버전)
```
48V ──[스위치]──[인덕터]──[캐패시터]── 5V 출력
        │                      │
        └── 제어 IC가 스위치를   │
            빠르게 ON/OFF하여    │
            평균 전압을 5V로 맞춤 │
```

1. 스위치 ON → 인덕터에 에너지 저장
2. 스위치 OFF → 인덕터가 저장된 에너지를 출력으로 방출
3. 이 과정을 수백 kHz로 반복
4. 캐패시터가 출력을 매끄럽게 유지

### 핵심 스펙 선정 시 고려사항 (벅 ②: 5V용)
```
입력 전압 범위: 12V ~ 19V (벅①의 출력)
출력 전압: 5V
출력 전류: 최소 2A (MCU + 센서 + 여유)
효율: 90% 이상 목표

→ 입력이 15V밖에 안 되니까, 아주 흔한 벅 모듈로 충분!
   (예: LM2596, MP1584, 아두이노 벅 모듈 등)
```

## 4. LDO (Low Dropout Regulator) — 5V → 3.3V

### 뭐하는 놈인가?
- 전압을 **조금만** 낮추는 간단한 레귤레이터
- 벅 컨버터보다 **노이즈가 적음** → 민감한 IC에 적합

### 왜 벅 컨버터 대신 LDO?
```
벅 컨버터: 효율 높음, but 스위칭 노이즈 있음
LDO:       효율 낮음, but 깨끗한 전원 출력

5V → 3.3V는 차이가 1.7V밖에 안 됨 → LDO 효율도 괜찮음
효율 = 3.3V / 5V = 66% (이 경우는 OK)
```

### 핵심 스펙
```
입력: 5V
출력: 3.3V
전류: 최소 1A (STM32 300mA + ESP32 300mA + 센서 + 여유)
드롭아웃 전압: < 0.5V (5V - 3.3V = 1.7V이므로 충분)
```

## 5. 디커플링 캐패시터 (Decoupling Capacitor)

### 이것이 왜 모든 IC 옆에 있는가?

IC(칩)가 갑자기 전류를 많이 필요로 하면:
```
전원 라인이 멀면 → 순간적으로 전압이 떨어짐 (전압 강하)
                 → IC가 오동작 할 수 있음
```

디커플링 캡 = **IC 옆에 놓는 작은 배터리 역할**
```
IC가 갑자기 전류 필요 → 바로 옆 캐패시터에서 즉시 공급
                      → 전압 강하 방지
```

### 규칙
- **모든 IC의 VCC-GND 핀 옆에** 100nF (0.1μF) 세라믹 캐패시터 배치
- 가능하면 **가까이** (짧은 배선으로)
- 전원 입력부에는 큰 캡도 추가 (10μF ~ 100μF)

```
[3.3V] ──┬── [100nF] ── [GND]    ← IC 바로 옆
         │
         └── [STM32 VCC 핀]
```

## 6. 접지 (GND, Ground)

### 왜 중요한가?
- GND = 전기 회로의 **기준점** (0V)
- 모든 전압은 GND 기준으로 측정됨
- **모든 부품의 GND는 하나로 연결**되어야 함

### AR Walker에서의 GND
```
48V 배터리 GND ─┬─ Elmo GND
                ├─ 벅 컨버터 GND
                ├─ STM32 GND
                ├─ ESP32 GND
                ├─ Jetson GND
                └─ 센서 GND
```

### 주의사항
- 전력용 GND (모터)와 신호용 GND (MCU)는 **PCB에서 분리 후 한 점에서 연결** (Star Ground)
- 그래야 모터의 큰 전류가 MCU의 섬세한 신호에 노이즈를 주지 않음

## 연습 문제

### 문제 1: 벅 컨버터 효율
> 벅 컨버터 ①의 효율이 90%이고, 15V에서 65W를 출력해야 한다.
> 48V 입력에서 몇 A의 전류가 필요한가?

<details>
<summary>정답 보기</summary>

```
출력 전력 = 65W
입력 전력 = 65W / 0.90 = 72.2W (효율 90%)
입력 전류 = 72.2W / 48V = 1.5A
```
</details>

### 문제 2: 배터리 수명
> 48V 20Ah 배터리를 사용한다.
> 시스템이 평균 500W를 사용하면 몇 시간 쓸 수 있나?

<details>
<summary>정답 보기</summary>

```
배터리 에너지 = 48V × 20Ah = 960Wh
사용 시간 = 960Wh / 500W = 1.92시간 ≈ 약 2시간
```
</details>

### 문제 3: LDO 발열
> LDO가 5V → 3.3V로 변환하고, 출력 전류가 1A일 때
> LDO에서 발생하는 열은 몇 W인가?

<details>
<summary>정답 보기</summary>

```
열 = (입력전압 - 출력전압) × 전류
   = (5V - 3.3V) × 1A
   = 1.7W

→ 1.7W면 방열이 필요할 수 있음 (손으로 만지면 뜨거움)
```
</details>

## 8. 배터리 시스템 (24V × 2 직렬)

### 구성

```
[24V 배터리 ①]──직렬──[24V 배터리 ②] = 48V
  (자체 BMS 내장)       (자체 BMS 내장)
```

24V 리튬 배터리 2개를 직렬 연결하여 48V를 만든다.

### 왜 48V 1개가 아니라 24V × 2개?
| | 48V 1개 | 24V × 2개 |
|--|---------|----------|
| 가격 | 비쌈 | **저렴** (24V 제품 선택지 많음) |
| 교체 | 전체 교체 | **하나만 교체** 가능 |
| BMS | 13S 필요 (복잡) | **각각 6~7S** (자체 BMS 내장) |
| 구매처 | 전문 업체 | 전동킥보드/전동공구용 등 **흔함** |
| 밸런싱 | 13셀 밸런싱 | 각 팩 내부에서 자체 밸런싱 |

### 직렬 연결 시 주의사항

```
⚠️ 핵심 규칙:
  1. 두 배터리의 용량(Ah)이 같아야 함
  2. 두 배터리의 충전 상태가 비슷해야 함
  3. 같은 제조사, 같은 모델 사용 권장
  4. 각 배터리의 GND가 공통이 아님! (직렬이니까)

연결:
  배터리①(+) ── 시스템 48V(+)
  배터리①(-) ── 배터리②(+)    ← 중간 연결점 (24V 지점)
  배터리②(-) ── 시스템 GND
```

### 용량 선정

```
총 전력 소비 (최악의 경우):
  모터:    ~500W (평균, 보행 시)
  Jetson:  ~40W
  캡처카드: ~20W
  기타:    ~12W
  합계:    ~572W

목표 운용 시간: 1시간으로 가정

필요한 배터리 에너지:
  572W × 1h = 572Wh

48V 기준 필요 용량:
  572Wh / 48V = 11.9Ah → 최소 12Ah

→ 24V 15~20Ah 배터리 2개 권장 (여유 포함)
  에너지: 48V × 20Ah = 960Wh → 약 1.7시간 운용
```

### 배터리 잔량 모니터링 — BMS 읽기

#### 전체 구조
```
[배터리① BMS] ──UART/SMBus──→ [STM32]
[배터리② BMS] ──UART/SMBus──→ [STM32]
                                  │
                          잔량 계산 & GUI 전송
                                  │
[저항 분압] ──ADC──→ [STM32]     (백업: 총 전압 측정)
```

#### BMS 통신 (메인)
```
대부분의 스마트 BMS는 통신 포트를 제공:
  - UART (가장 흔함): TX/RX 2선으로 데이터 수신
  - SMBus/I2C: 표준 배터리 통신
  - 블루투스: 앱 모니터링용 (직접 연결 불가)

BMS에서 읽을 수 있는 정보:
  ✓ 총 전압
  ✓ 잔량 (SOC, %)
  ✓ 전류 (충전/방전)
  ✓ 각 셀 전압 (밸런싱 상태)
  ✓ 온도
  ✓ 보호 상태 (과충전/과방전/과전류/단락)
```

#### UART BMS 프로토콜 (일반적인 형식)
```
STM32 → BMS:  [시작바이트] [명령] [길이] [체크섬]
BMS → STM32:  [시작바이트] [명령] [데이터...] [체크섬]

예 (일반적인 스마트 BMS):
  요청: 0xDD 0xA5 0x03 0x00 0xFF 0xFD 0x77
  응답: 0xDD 0x03 [총전압 2B] [전류 2B] [잔량 2B] [온도...] [체크섬]
```

#### 전압 분압 ADC (백업)
```
48V를 직접 STM32 ADC(0~3.3V)에 넣을 수 없음
→ 저항 분압기로 전압을 낮춤

[48V(+)] ──[R1: 150kΩ]──┬──→ [STM32 ADC 핀]
                          │
                        [R2: 10kΩ]
                          │
[GND] ────────────────────┘

ADC 전압 = 48V × R2 / (R1 + R2)
         = 48V × 10k / 160k
         = 3.0V ← ADC 범위 안!

전압 → 잔량 변환 (리튬 기준):
  만충:  2 × 29.4V = 58.8V → ADC 3.68V (주의: 3.3V 초과!)
  공칭:  2 × 25.2V = 50.4V → ADC 3.15V
  방전:  2 × 21.0V = 42.0V → ADC 2.63V

⚠️ 만충 시 3.3V 초과 가능 → R1을 더 크게 조정 필요!
  R1 = 180kΩ, R2 = 10kΩ 으로 변경하면:
  58.8V × 10/190 = 3.09V ← 안전!
```

#### STM32 구현 (코드 개요)
```c
// BMS UART 읽기 (메인)
void read_bms_data(UART_HandleTypeDef *huart, BatteryData *bat) {
    uint8_t request[] = {0xDD, 0xA5, 0x03, 0x00, 0xFF, 0xFD, 0x77};
    HAL_UART_Transmit(huart, request, 7, 100);
    HAL_UART_Receive(huart, rx_buf, 32, 100);

    bat->voltage = (rx_buf[4] << 8 | rx_buf[5]) * 0.01f;  // 전압
    bat->current = (rx_buf[6] << 8 | rx_buf[7]) * 0.01f;  // 전류
    bat->soc     = rx_buf[23];                              // 잔량 %
}

// ADC 전압 읽기 (백업)
float read_battery_voltage_adc(void) {
    uint16_t adc_val = HAL_ADC_GetValue(&hadc1);
    float adc_voltage = adc_val * 3.3f / 65536.0f;  // 16비트 ADC
    float battery_voltage = adc_voltage * (180.0f + 10.0f) / 10.0f;  // 역산
    return battery_voltage;
}
```

### 전체 전원 시스템 (최종)

```
[24V 배터리①]──직렬──[24V 배터리②] = 48V
  │ (BMS UART)          │ (BMS UART)
  │                      │
  └──────┬───────────────┘
         │
    [48V 메인 라인]──[퓨즈]──[역극성 보호]──[TVS 다이오드]
         │
         ├──→ [Elmo Gold Twitter × 2~4] → [모터] (48V 직접)
         │
         ├──→ [전압 분압] → [STM32 ADC] (배터리 전압 백업 측정)
         │
         └──→ [벅 ①: 48V→15V, 80W]
                   │
                   ├──→ Jetson Orin Nano Super (~40W)
                   ├──→ 캡처카드 + 카메라 (~20W)
                   │
                   └──→ [벅 ②: 15V→5V]
                              │
                              ├──→ EBIMU RF 리시버 (5V)
                              │
                              └──→ [LDO: 5V→3.3V]
                                        │
                                        ├──→ STM32H743
                                        ├──→ ESP32-S3
                                        ├──→ INA828 × 4
                                        └──→ CAN 트랜시버
```

| 변환 단계 | 방식 | 입력 | 출력 | 전력 | 대상 |
|-----------|------|------|------|------|------|
| 직접 | 없음 | 48V | 48V | ~1000W | Elmo 모터 드라이버 |
| 벅 ① | 스위칭 | 48V | **15V** | **65W+** | Jetson, 캡처카드, 벅② 입력 |
| 벅 ② | 스위칭 | **15V** | 5V | ~6W | LDO 입력 |
| LDO | 리니어 | 5V | 3.3V | ~2.5W | STM32, ESP32, 센서 |

**배터리: 24V × 2직렬 = 48V**
**전원 직렬 구조: 48V → 15V → 5V → 3.3V**

**기억할 것:**
- 배터리: **24V 15~20Ah × 2직렬**, 각각 자체 BMS 내장
- 잔량 모니터링: **BMS UART 통신** (메인) + 전압 분압 ADC (백업)
- 벅 ①만 48V 고입력 → **대전력 부품** 필요 (65W+)
- 벅 ②는 15V 입력 → **흔한 저가 부품**으로 OK
- 보호: **퓨즈 + 역극성 보호 + TVS 다이오드**
- 모든 IC 옆에 **디커플링 캡** (100nF)
- 전력용 GND와 신호용 GND **분리 후 한 점 연결**
