# Step 2: AR Walker 전원 시스템 이해

> 우리 프로젝트의 전기가 어디서 오고, 어디로 가는지 이해하자.

---

## 1. 전원 트리 (Power Tree)

AR Walker의 전원은 **48V 배터리 하나**에서 시작해서 여러 갈래로 나뉜다:

```
[48V 리튬 배터리]
    │
    ├──→ [Elmo Gold Twitter] → [모터] (48V 직접 사용)
    │
    ├──→ [벅 컨버터 ①] (48V → 15V, 65W 이상)
    │         │
    │         └──→ [15V 출력] (12~19V 범위)
    │                 │
    │                 ├──→ [Jetson Orin Nano Super] (15V, ~65W)
    │                 │
    │                 └──→ [캡처카드 + 카메라] (12~19V)
    │
    ├──→ [벅 컨버터 ②] (48V → 5V)
    │         │
    │         └──→ [5V 출력]
    │                 │
    │                 └──→ [LDO (Low Dropout Regulator)]
    │                          │
    │                          └──→ [3.3V 출력]
    │                                  │
    │                                  ├──→ [STM32H743]
    │                                  ├──→ [ESP32-S3]
    │                                  ├──→ [INA828 x4]
    │                                  └──→ [CAN 트랜시버]
    │
    └──→ [EBIMU RF 리시버] (자체 전원 또는 5V/3.3V)
```

> **주의**: 벅 컨버터가 **2개** 필요하다!
> - ① 48V → 15V: Jetson + 캡처카드용 (고전력, 65W+)
> - ② 48V → 5V: MCU/센서용 (저전력, ~5W)

---

## 2. 벅 컨버터 ① — 48V → 15V (Jetson + 캡처카드)

### 이것이 가장 큰 전력 소비자!

Jetson Orin Nano Super는 **5V가 아니라 12~19V** 전원이 필요하다.
캡처카드(카메라용)도 같은 12~19V 범위로 전원을 공급해야 한다.

```
요구사항:
  입력: 48V (배터리)
  출력: 15V (12~19V 범위 내)
  전력: 65W 이상 (Jetson ~40W + 캡처카드 + 여유)
  전류: 65W / 15V = 4.3A 이상

→ 대전력 벅 컨버터 필요! (일반 소형 벅으로는 부족)
```

### 왜 15V인가?
```
Jetson Orin Nano Super 입력 범위: 12~19V
캡처카드 입력 범위: 12~19V

15V = 범위의 중간값 → 가장 안정적
  - 12V에 너무 가까우면: 부하 시 전압 강하로 12V 밑으로 떨어질 수 있음
  - 19V에 너무 가까우면: 발열 증가
```

## 3. 벅 컨버터 ② — 48V → 5V (MCU/센서)

### 뭐하는 놈인가?
- 높은 전압을 **낮은 전압으로 변환**하는 장치
- "스위칭" 방식이라 **효율이 높음** (보통 85~95%)

### 왜 필요한가?
- STM32, ESP32는 3.3V에서 동작
- 48V를 직접 넣으면 **즉시 파괴**됨
- 48V → 3.3V를 한 번에 하면 효율이 떨어짐
- 그래서 **48V → 5V → 3.3V** 두 단계로 변환

### 동작 원리 (간단 버전)
```
48V ──[스위치]──[인덕터]──[캐패시터]── 5V 출력
        │                      │
        └── 제어 IC가 스위치를   │
            빠르게 ON/OFF하여    │
            평균 전압을 5V로 맞춤 │
```

1. 스위치 ON → 인덕터에 에너지 저장
2. 스위치 OFF → 인덕터가 저장된 에너지를 출력으로 방출
3. 이 과정을 수백 kHz로 반복
4. 캐패시터가 출력을 매끄럽게 유지

### 핵심 스펙 선정 시 고려사항 (벅 ②: 5V용)
```
입력 전압 범위: 42V ~ 54V (48V 리튬 배터리 충방전 범위)
출력 전압: 5V
출력 전류: 최소 2A (MCU + 센서 + 여유)
효율: 90% 이상 목표
```

## 4. LDO (Low Dropout Regulator) — 5V → 3.3V

### 뭐하는 놈인가?
- 전압을 **조금만** 낮추는 간단한 레귤레이터
- 벅 컨버터보다 **노이즈가 적음** → 민감한 IC에 적합

### 왜 벅 컨버터 대신 LDO?
```
벅 컨버터: 효율 높음, but 스위칭 노이즈 있음
LDO:       효율 낮음, but 깨끗한 전원 출력

5V → 3.3V는 차이가 1.7V밖에 안 됨 → LDO 효율도 괜찮음
효율 = 3.3V / 5V = 66% (이 경우는 OK)
```

### 핵심 스펙
```
입력: 5V
출력: 3.3V
전류: 최소 1A (STM32 300mA + ESP32 300mA + 센서 + 여유)
드롭아웃 전압: < 0.5V (5V - 3.3V = 1.7V이므로 충분)
```

---

## 5. 디커플링 캐패시터 (Decoupling Capacitor)

### 이것이 왜 모든 IC 옆에 있는가?

IC(칩)가 갑자기 전류를 많이 필요로 하면:
```
전원 라인이 멀면 → 순간적으로 전압이 떨어짐 (전압 강하)
                 → IC가 오동작 할 수 있음
```

디커플링 캡 = **IC 옆에 놓는 작은 배터리 역할**
```
IC가 갑자기 전류 필요 → 바로 옆 캐패시터에서 즉시 공급
                      → 전압 강하 방지
```

### 규칙
- **모든 IC의 VCC-GND 핀 옆에** 100nF (0.1μF) 세라믹 캐패시터 배치
- 가능하면 **가까이** (짧은 배선으로)
- 전원 입력부에는 큰 캡도 추가 (10μF ~ 100μF)

```
[3.3V] ──┬── [100nF] ── [GND]    ← IC 바로 옆
         │
         └── [STM32 VCC 핀]
```

## 6. 접지 (GND, Ground)

### 왜 중요한가?
- GND = 전기 회로의 **기준점** (0V)
- 모든 전압은 GND 기준으로 측정됨
- **모든 부품의 GND는 하나로 연결**되어야 함

### AR Walker에서의 GND
```
48V 배터리 GND ─┬─ Elmo GND
                ├─ 벅 컨버터 GND
                ├─ STM32 GND
                ├─ ESP32 GND
                ├─ Jetson GND
                └─ 센서 GND
```

### 주의사항
- 전력용 GND (모터)와 신호용 GND (MCU)는 **PCB에서 분리 후 한 점에서 연결** (Star Ground)
- 그래야 모터의 큰 전류가 MCU의 섬세한 신호에 노이즈를 주지 않음

## 7. 전력 예산 (Power Budget) 계산

AR Walker 전체 시스템의 전력을 계산해보자:

### 48V 레일 (배터리 직접)
| 부품 | 전압 | 전류 (최대) | 전력 |
|------|------|------------|------|
| Elmo + 모터 (x2~4) | 48V | 10~20A | 480~960W |
| 벅 컨버터 ① 입력 (15V용) | 48V | ~1.5A | ~72W |
| 벅 컨버터 ② 입력 (5V용) | 48V | ~0.2A | ~10W |
| **합계** | | | **~560~1040W** |

### 15V 레일 (벅 컨버터 ① 출력) ★ 가장 큰 보조 전력
| 부품 | 전압 | 전류 (최대) | 전력 |
|------|------|------------|------|
| Jetson Orin Nano Super | 15V | ~2.7A | ~40W |
| 캡처카드 + 카메라 | 15V | ~1.3A | ~20W |
| **합계** | | | **~60W** |
| **벅 컨버터 ① 스펙** | | **65W 이상** | **4.3A 이상** |

### 5V 레일 (벅 컨버터 ② 출력)
| 부품 | 전압 | 전류 (최대) | 전력 |
|------|------|------------|------|
| LDO 입력 → 3.3V | 5V | 1A | 5W |
| 기타 5V 소비자 | 5V | 0.5A | 2.5W |
| **합계** | | | **~7.5W** |

### 3.3V 레일 (LDO 출력)
| 부품 | 전압 | 전류 (최대) | 전력 |
|------|------|------------|------|
| STM32H743 | 3.3V | 300mA | 1W |
| ESP32-S3 | 3.3V | 300mA | 1W |
| INA828 x4 | 3.3V | 20mA | 0.07W |

## 연습 문제

### 문제 1: 벅 컨버터 효율
> 벅 컨버터 ①의 효율이 90%이고, 15V에서 65W를 출력해야 한다.
> 48V 입력에서 몇 A의 전류가 필요한가?

<details>
<summary>정답 보기</summary>

```
출력 전력 = 65W
입력 전력 = 65W / 0.90 = 72.2W (효율 90%)
입력 전류 = 72.2W / 48V = 1.5A
```
</details>

### 문제 2: 배터리 수명
> 48V 20Ah 배터리를 사용한다.
> 시스템이 평균 500W를 사용하면 몇 시간 쓸 수 있나?

<details>
<summary>정답 보기</summary>

```
배터리 에너지 = 48V × 20Ah = 960Wh
사용 시간 = 960Wh / 500W = 1.92시간 ≈ 약 2시간
```
</details>

### 문제 3: LDO 발열
> LDO가 5V → 3.3V로 변환하고, 출력 전류가 1A일 때
> LDO에서 발생하는 열은 몇 W인가?

<details>
<summary>정답 보기</summary>

```
열 = (입력전압 - 출력전압) × 전류
   = (5V - 3.3V) × 1A
   = 1.7W

→ 1.7W면 방열이 필요할 수 있음 (손으로 만지면 뜨거움)
```
</details>

## 핵심 요약

| 변환 단계 | 방식 | 입력 | 출력 | 전력 | 대상 |
|-----------|------|------|------|------|------|
| 직접 | 없음 | 48V | 48V | ~1000W | Elmo 모터 드라이버 |
| 벅 ① | 스위칭 | 48V | **15V** | **65W+** | Jetson, 캡처카드 |
| 벅 ② | 스위칭 | 48V | 5V | ~8W | LDO 입력 |
| LDO | 리니어 | 5V | 3.3V | ~2.5W | STM32, ESP32, 센서 |

**기억할 것:**
- 벅 컨버터가 **2개** 필요 (15V + 5V 각각)
- 15V 벅은 **65W 이상** 대전력 제품 선정 필수
- 모든 IC 옆에 **디커플링 캡** (100nF)
- 전력용 GND와 신호용 GND **분리 후 한 점 연결**
- 전력 예산을 미리 계산해서 부품 선정
