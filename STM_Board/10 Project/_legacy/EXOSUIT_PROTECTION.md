# Exosuit 보드 보호 회로 설계 — 최종 확정본

> **배경**: 케이블 드리븐 외골격의 반복적 pull-release → 수십 A 전류 급변 → 보드 반복 파손
> 4개 전문 에이전트 병렬 분석 + 교차검증으로 수치 근거 확정

---

## 왜 보드가 타는가 — 수치 분석

### 메커니즘 1: 재생 에너지 전압 폭주 (가장 위험)

```
케이블 릴리즈 순간:
  케이블 탄성에너지 = F×x/2 = 100N × 5mm / 2 = 0.25J/모터
  4개 동시 릴리즈: E_total = 1J

  배터리 BMS가 역전류 거부할 때 에너지 행선지:
  → 버스 캐패시터 충전

  C_bus = 100μF (보호 없을 때):
  V_peak = √(2×1J / 100μF + 24²) = √(20,000 + 576) = 143V  ← 즉사

  C_bus = 10,000μF (보호 후):
  V_peak = √(2×1J / 10,000μF + 24²) = √(200 + 576) = 27.9V ← 안전
```

### 메커니즘 2: Ground Bounce → CAN 트랜시버 TXD/RXD 파손

```
모터 리턴 전류 급변 + PCB 그라운드 인덕턴스:
  V_bounce = L × dI/dt = 50nH × 50A/μs = 2.5V

TJA1051 TXD/RXD 핀 (VIO 기준):
  절대최대정격: VIO ± 0.3V
  2.5V bounce → 즉시 초과 → 트랜시버 파손

  이것이 CAN 트랜시버가 주기적으로 죽는 원인
```

### 메커니즘 3: L·dI/dt 버스 스파이크

```
케이블 + PCB 배선 인덕턴스 총계 ≈ 700nH ~ 2μH
극단 케이스: V = 2μH × 2A/μs = 4V → 버스 최대 29.2V
→ 재생 에너지보다 작지만 고주파 반복 → 소자 열화 가속
```

## PCB 레이아웃 규칙 (보호 관점)

### 1. 고전류 루프 면적 최소화 (인덕턴스 감소)

```
[잘못된 배치] 배터리 커넥터 → 트레이스 100mm → 캐패시터 → 모터
→ 루프 면적 크면 L 증가 → V_spike 증가

[올바른 배치] 배터리 커넥터 옆 3mm 이내에 벌크 캐패시터 배치
→ BUS_BULK ↔ 배터리 커넥터 사이 트레이스 최단
→ 캐패시터 GND via는 패드 직결, 별도 트레이스 없이
```

### 2. 모터 전원 리턴 경로 전용화

```
모터 커넥터 GND 핀
     ↓ (via 4개 이상)
Layer 2 PGND 플레인
     ↓ (넓은 Pour)
배터리 GND 커넥터
→ 이 경로에 MCU, 센서 GND 비아 없음
```

### 3. 격리 경계선 (ISO1050)

```
PCB Layer 1~4 모두:
  MCU 영역 (AGND) │ 솔더마스크 갭 2mm │ CAN 버스 영역 (PGND)
ISO1050 패키지가 이 경계를 가로지름
ISO1050 아래 Layer 2: AGND와 PGND 사이 갭 (via 없음)
```

### 4. 제동 저항 배치

```
IPB025N10N3 → PCB 모서리 or 외부 방열판 연결
저항 HS50 3Ω → PCB 외부 별도 방열판 (50W 소산)
MOSFET 방열: TO-263 패드 → 2oz 동박 방열 패드 최소 4cm²
```

### 5. CAN 보호 소자 배치 순서

```
커넥터 핀에서 거리:
3mm: SM712-02HTG TVS
8mm: ACM2012-900 CMC
13mm: 22Ω 댐핑 저항
18mm: ISO1050 버스측 핀
→ 격리 장벽 →
23mm: PRTR5V0U2X ESD
28mm: 33Ω MCU 측 저항
35mm: STM32H743 FDCAN 핀
```
